# Welcome to the Party! 🎉

Hey there, awesome human!  
Thanks for thinking about contributing to this project. Whether you’re here to squash bugs, add features, correct typos, or drop some wisdom, we’re pumped to have you on board.

---

## 🚦 Quick Start Checklist

1. 🍴 **Fork** this repo.
2. 🛠️ **Create a branch**:  
   `git checkout -b feature/my-cool-thing`
3. 💻 **Make your changes** (and make ‘em shine!).
4. ✅ **Test** to make sure you didn’t set anything on fire (unless it’s a `🐉 PANT ON FIRE` issue).
5. 📬 **Submit a pull request** – and fill out the PR template.
6. 🎯 **Tag your PR/issue** with a priority label (see below).

---

## 🔥 Priority Labels: How Screwed Are We?

| Label                | Meaning                                                                 |
|----------------------|-------------------------------------------------------------------------|
| 🐉 **PANT ON FIRE**  | “Evacuate the building!” – Critical, needs immediate attention.         |
| 🚨 **RUHROH**        | “Scooby Snack shortage!” – Serious, but not apocalyptic.                |
| 🤔 **HMMM...**       | “Mild confusion detected.” – Worth fixing, but not urgent.              |
| 😴 **MEH**           | “Nap time.” – Cosmetic, low-priority, or just for fun.                  |

*Please pick the label that best fits your PR or issue. If you’re not sure, take your best guess – we’ll help!*

## 📝 How to File an Issue

- **Search first!** Someone else might have already reported your brilliant observation.
- Use our [issue template](.github/ISSUE_TEMPLATE) and fill out the W5H checklist:
  - **Who** is affected?
  - **What** happened (or didn’t)?
  - **When** does it happen?
  - **Where** in the project?
  - **Why** does it matter?
  - **How** do you think it could be fixed (if you have ideas)?

Screenshots, code snippets, and GIFs are always welcome!

---

## 🤝 Pull Requests: The Secret Sauce

- Use our [PR template](.github/pull_request_template.md) – the more details, the better!
- Reference related issues with `Closes #issue-number`.
- Keep your changes focused – one thing per PR is best.
- Be ready for friendly feedback. We’re here to help, not to judge!

---

## 🧹 Code Style & Best Practices

- Match the existing code style (when in doubt, copy-paste and tweak).
- Write tests for new features or bug fixes.
- Document anything that isn’t obvious.
- Run all tests before you push.
